# Master Systemic Checklist

Use before merge (or before release). If any answer creates hesitation → fix required.

## Questions

1. Does this screen feel **calm**?
2. Is there **one clear primary action**?
3. Does **every action confirm clearly** (success or failure)?
4. Is anything **unnecessary** (no clarity, action, or meaning)?
5. Does **layout stay stable** during load?
6. Is **feedback immediate** (or loading shown)?
7. Is there any **moment of doubt** (e.g. "did it work?")?
8. Does anything feel **sloppy** or **unfinished**?
9. Would I **trust this with real money**?

## Prohibited

- Adding UI to justify complexity
- Keeping controls "just in case"
- Multiple competing primary CTAs
- Silent failures
- Hidden state changes
- Technical jargon in user messaging
- Flicker during updates
- Excessive modal stacking

## Completion criteria

- UI feels calm and intentional
- No silent feedback gaps
- No unnecessary UI clutter
- Actions produce visible consequence
- Tasks require minimal friction
- App feels stable and trustworthy
- Nothing feels "off" or sloppy
